设计模式学习记录-01

发表于 2019-05-27 | 更新于 2019-05-29 | 分类于 设计模式

学之前在想的问题

感受设计演变过程中所蕴含的大智慧,体会乐与怒的程序人生中值得回味的一幕.

写下这篇博客的时候,我手头还有一个游戏项目没有写完,而这并不能妨碍我思考,不妨碍我写下这篇博客.在正式学习之前,我一直在思考一些问题.

  • 大学这两年我学了什么?
  • 为什么要学习设计模式?
  • 学习设计模式是为了什么?
  • 设计模式的书好厚,完全看不下去怎么破?
  • 应该什么时候学设计模式呢?

1. 大学这两年我学了什么?

从最初因为喜欢编程选择转专业到现在,总感觉自己学了很多,又似乎学的很少,似乎自己的编程能力也有点了,但似乎始终少点什么?

1.1 大一

初入大学,报着努力学习的远大理想,然而现实是作业不到最后不写,游戏每天打打,似乎过得很充实的样子,着实让自己体验了什么叫充实而又空虚着.

学习了各种基础课程,但又学的不怎么好,总认为自己学了很多,回想起来,似乎早已忘了很多,并没有真正吸收.

1.2 大二

这段时间,把编程作为游戏来玩,体验到了前所未有的成就感,这是以前游戏无法带给我的一种感觉,简直不要太爽,然而坐时间久了之后,身体似乎也被自己搞出了问题,每次起身腰都好痛,提醒了自己坐的时候坐姿要摆正,不能乱扭.

1.3 总结

  1. 学了一些基础课程及 C 语言．
  2. 学习了 Java && C# && JavaScript,学习了皮毛一般的面向对象思想．
  3. 学习了数据库系统的整体设计及 DBMS 的操作．
  4. 自学了 Unity 3D 引擎开发游戏的基础及部分进阶技术
  5. 总之，要走的路还很长

1.为什么要学习设计模式？

最初的自己也总是在想,设计模式到底是因为什么而被研究创造出来的,一本超级厚的’设计模式’完全看不下去,又厚又难懂,总是看不了多少就坚持不下去了.

作为软工的学生,全称可是软件工程,作为工程,就不能以编程能力来衡量自己学的好不好,而应该以架构+编程能力这样的指标来衡量自己.

代码只有打多了才会想的多,才能更深入去了解其架构,其设计方式,倘若认为自己学会了某个框架的使用就足够了,就很强了,那可能是见到的太少,认知被限定了.

私以为编程分为以下几个境界:

  • 学会语法
  • 会简单的逻辑编写
  • 学会一定量算法
  • 学会一个框架的使用
  • 了解框架的原理
  • 自己可以搭建框架
  • 对各种框架都有所涉猎
  • 不再拘泥于框架

以上仅为个人感觉,可能境界上不足,只能想到这些.而要达到最高境界,对于设计模式肯定要了解,并且可以比较轻松的说出一些耳熟能详的模式,还可以将设计模式进行结合,进行更好的编程架构.

2.设计模式的书好厚,看不下去怎么办?

其实很简单,多打一些项目工程,并且多去思考,当这些过程经历多了,自然会有去认真学习设计模式的动力.当然,学好之后可以装十三也是很好的动力.

3.应该什么时候学设计模式?

这个问题在上面也也已经回答了,当有学习设计模式的念头与需求时,就赶紧学吧,这将是效率最高的时候!

写在最后的话

私以为自己编程能力还差很多,只是比周围人强了那么一点点,代码的世界里没有最强,只有更强,沉下心来学,基础打扎实,为以后做项目打好地基!
